## 2. Tracker réglementaire — Pipeline

### Architecture
```
earlybrief-platform (Scaleway)
→ GET /api/v1/articles (published_at, score, source)
→ tracker-admin (early-brief.com/outils/tracker-admin/)
→ validation humaine (approve/reject par carte)
→ describe-tracker.js (1. reformulation titre, 2. classification type+destination)
→ admin-server.cjs route par destination
→ tracker-data.json (TEXTE/DECISION/CONSULTATION)
  + autres-publications-data.json (RAPPORT/INTELLIGENCE)
→ GitHub API commit (tracker-data.json + autres-publications-data.json)
→ Netlify deploy automatique
```

## 3. Page "Autres publications"

### Architecture
- URL : `/outils/autres-publications/`
- Fichier : `public/outils/autres-publications/index.html`
- Données : `public/outils/autres-publications/autres-publications-data.json`
- Contenu : discours, rapports, positions, MoU, Q&A — hors textes applicables
- Colonnes : Date, Source, Type (badge RAPPORT/INTELLIGENCE), Titre, Lien
- Filtres : Mois, Source, Type
- Export CSV
- Accessible via `/ressources/` > Outils interactifs (pas dans la nav principale)

### Routage admin-server.cjs (POST /api/tracker/approve)
```
entries.filter(e => e.destination === 'tracker')   → tracker-data.json
entries.filter(e => e.destination !== 'tracker')   → autres-publications-data.json
```
- Chaque entrée porte `type` et `destination` assignés par describe-tracker.js
- L'opérateur peut modifier la destination manuellement via un select dans tracker-admin avant validation
- Réponse : `{added_tracker: N, added_actualites: N}`
- Commit message : `"tracker+autres-publications: mise à jour DD/MM/YYYY"`

## 6. Descriptions automatiques (describe-tracker.js)

### Netlify Function : `netlify/functions/describe-tracker.js`

**Appel 1 — Reformulation du titre** (Claude claude-sonnet-4-20250514, temperature 0, max_tokens 80)
- Prompt : reformulation factuelle du titre uniquement (pas du summary)
- Entrée : source + titre brut
- Sortie : titre reformulé en français, une phrase factuelle

**Appel 2 — Classification** (Claude claude-sonnet-4-20250514, temperature 0, max_tokens 50)
- Prompt système définit les 5 types + mapping destination
- Entrée : source + titre
- Sortie : JSON `{"type": "...", "destination": "..."}`
- Destination exacte : `"tracker"` ou `"autres-publications"`

**Retour combiné** : `{description, type, destination}`

**Fallbacks** :
- Si erreur API appel 1 → titre brut
- Si erreur API appel 2 → `{type: 'INTELLIGENCE', destination: 'autres-publications'}`

**Traitement** : batch de 10 candidats, 500ms entre batches (rate limit Anthropic 50 req/min)

**Variable d'environnement** : `ANTHROPIC_API_KEY` configurée dans Netlify

### Chaîne complète de filtres (dans l'ordre)
1. `isTrackerSource` : source dans PRIORITY_SOURCES (tout) ou OTHER_SOURCES (score >= 30)
2. `isOfficialDomain` : URL correspond à un des 30 domaines whitelistés
3. `isNotGeneric` : titre pas dans GENERIC_TITLES
4. `hasMeaningfulTitle` : titre > 15 caractères
5. `afterCutoff` : published_at >= 2026-01-01
6. `isNew` : URL pas déjà dans tracker-data.json ou autres-publications-data.json
7. `isNotGoogleNews` : URL ne contient pas news.google.com
